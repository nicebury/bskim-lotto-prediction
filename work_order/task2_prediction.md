# 로또 번호 예측 시스템 작업지시서

## 1. 프로젝트 개요

역대 로또 당첨번호 데이터(SQLite DB)를 기반으로, **앙상블 스코어링 + 몬테카를로 시뮬레이션**을 조합하여 다음 회차 로또 번호 6개 + 보너스 1개를 추천하는 Python 모듈을 개발한다.

> ⚠️ 로또는 완전한 랜덤이므로 예측은 불가능하다. 본 시스템은 통계적 재미 목적이다.

---

## 2. 기존 시스템 정보

### 2.1 DB 스키마

```sql
CREATE TABLE lotto_results (
    round_no            INTEGER PRIMARY KEY,
    draw_date           TEXT    NOT NULL,
    num1                INTEGER NOT NULL,
    num2                INTEGER NOT NULL,
    num3                INTEGER NOT NULL,
    num4                INTEGER NOT NULL,
    num5                INTEGER NOT NULL,
    num6                INTEGER NOT NULL,
    bonus               INTEGER NOT NULL,
    total_sell_amount   INTEGER,
    first_win_amount    INTEGER,
    first_winner_count  INTEGER,
    first_accum_amount  INTEGER,
    created_at          TEXT    NOT NULL
);
```

- 번호 범위: 1~45
- 6개 당첨번호(num1~num6)는 오름차순 정렬되어 저장됨
- bonus는 보너스 번호

### 2.2 이미 구현된 기능
- 크롤러 (데이터 수집)
- 조회 화면

## 4. 모듈별 상세 명세

### 4.1 config.py — 설정값 관리

```python
# 분석 대상 번호 범위
NUMBER_RANGE = (1, 45)

# 핫넘버 분석 시 최근 회차 수
HOT_RECENT_ROUNDS = 20

# 몬테카를로 시뮬레이션 횟수
MONTE_CARLO_SIMULATIONS = 100_000

# 앙상블 가중치 (합계 = 1.0)
WEIGHTS = {
    "frequency": 0.25,     # 빈도 분석
    "delay": 0.25,         # 지연 번호
    "hot_cold": 0.30,      # 핫/콜드 (최근 트렌드 중시)
    "pattern": 0.20,       # 패턴 분석
}

# 최종 추천 세트 수
RECOMMEND_SETS = 5
```

### 4.2 frequency.py — 전체 빈도 분석

**목적**: 역대 전체 회차에서 각 번호(1~45)의 출현 빈도를 계산하고 점수화한다.

**로직**:
1. DB에서 전체 회차의 num1~num6를 조회한다.
2. 각 번호(1~45)의 총 출현 횟수를 카운트한다.
3. 보너스 번호는 별도로 카운트하되, 가중치 0.3을 적용한다. (보너스도 "선택된 번호"이므로 완전히 무시하지 않는다.)
4. **정규화**: 최소-최대 정규화로 0.0~1.0 범위의 점수로 변환한다.

**반환값**: `dict[int, float]` — 번호별 점수 (예: `{1: 0.72, 2: 0.65, ...}`)

**SQL 참고**:
```sql
-- 각 번호별 출현 횟수 집계 (num1~num6 + bonus 별도)
SELECT num, COUNT(*) as cnt FROM (
    SELECT num1 as num FROM lotto_results
    UNION ALL SELECT num2 FROM lotto_results
    UNION ALL SELECT num3 FROM lotto_results
    UNION ALL SELECT num4 FROM lotto_results
    UNION ALL SELECT num5 FROM lotto_results
    UNION ALL SELECT num6 FROM lotto_results
) GROUP BY num;

-- 보너스 번호 별도 집계
SELECT bonus as num, COUNT(*) as cnt FROM lotto_results GROUP BY bonus;
```

### 4.3 delay.py — 지연 번호 분석

**목적**: 각 번호가 마지막으로 출현한 이후 몇 회차가 지났는지 계산한다. 오래 안 나온 번호일수록 높은 점수를 부여한다.

**로직**:
1. DB에서 최신 회차 번호(max round_no)를 가져온다.
2. 각 번호(1~45)에 대해, 해당 번호가 마지막으로 출현한 회차를 조회한다. (num1~num6 중 하나라도 해당 번호인 가장 최근 회차)
3. `지연값 = 최신 회차 - 마지막 출현 회차`를 계산한다.
4. 지연값을 정규화(0.0~1.0)하여 점수로 변환한다.
5. **보정**: 극단적으로 오래 안 나온 번호(상위 5%)는 점수를 약간 깎는다. (너무 안 나오는 번호는 "원래 잘 안 나오는 번호"일 수 있으므로)

**반환값**: `dict[int, float]` — 번호별 점수

### 4.4 hot_cold.py — 핫/콜드 넘버 분석

**목적**: 최근 N회차(기본 20회)에서의 출현 빈도를 분석하여, 최근 트렌드를 반영한다.

**로직**:
1. 최근 `HOT_RECENT_ROUNDS`(20) 회차의 당첨번호를 조회한다.
2. 각 번호의 최근 출현 횟수를 카운트한다.
3. **가중 카운트**: 더 최근 회차일수록 높은 가중치를 부여한다.
   - 가중치 공식: `weight = 1.0 - (순서 / HOT_RECENT_ROUNDS) * 0.5`
   - 예: 가장 최근 회차 = 1.0, 20회차 전 = 0.5
4. 가중 카운트를 정규화(0.0~1.0)하여 점수화한다.

**반환값**: `dict[int, float]` — 번호별 점수

### 4.5 pattern.py — 패턴 분석

**목적**: 역대 당첨번호 조합의 구조적 패턴을 분석하여, "로또답지 않은 조합"을 걸러내는 필터 점수를 산출한다.

**분석 항목 및 로직**:

#### (a) 홀짝 비율
- 역대 당첨번호 6개의 홀수/짝수 비율 분포를 집계한다.
- 예: 홀3:짝3이 가장 많다면, 해당 비율에 높은 점수.
- 반환: 각 비율별 확률 (예: `{"3:3": 0.33, "4:2": 0.22, ...}`)

#### (b) 고저 비율
- 기준: 1~22 = 저번호, 23~45 = 고번호
- 홀짝과 동일하게 비율 분포를 집계한다.

#### (c) 연속번호 포함 여부
- 6개 번호 중 연속된 번호(예: 5,6 또는 12,13,14)가 포함된 비율을 집계한다.
- 역대 데이터에서 연속번호 포함 비율이 높다면, 연속번호를 포함한 조합에 가산점.

#### (d) 끝자리 분포
- 6개 번호의 끝자리(일의 자리) 분포를 분석한다.
- 끝자리가 다양할수록(중복이 적을수록) 높은 점수.

#### (e) 번호 합산 범위
- 6개 번호의 합산값 분포를 분석한다.
- 역대 가장 빈번한 합산 범위에 해당하면 높은 점수.

**반환값**: 패턴 기준표 — 몬테카를로에서 조합을 평가할 때 사용할 기준 딕셔너리.

```python
{
    "odd_even_dist": {"3:3": 0.33, "4:2": 0.22, ...},
    "high_low_dist": {"3:3": 0.30, "2:4": 0.25, ...},
    "consecutive_ratio": 0.65,  # 연속번호 포함 비율
    "sum_range": {"min": 100, "max": 175, "peak": 130},  # 합산 범위
    "tail_diversity_avg": 4.2   # 평균 끝자리 종류 수
}
```

## 5. 앙상블 스코어링 (ensemble.py)

**목적**: 4개 분석 모듈의 점수를 가중 합산하여 번호별 최종 점수를 산출한다.

**로직**:
1. 4개 모듈(frequency, delay, hot_cold)에서 각각 `dict[int, float]` (1~45 번호별 점수)를 받는다.
2. 각 모듈의 점수에 `config.WEIGHTS`의 가중치를 곱한다.
3. 번호별로 가중 합산하여 최종 점수를 계산한다.

```python
final_score[num] = (
    frequency_score[num] * WEIGHTS["frequency"] +
    delay_score[num] * WEIGHTS["delay"] +
    hot_cold_score[num] * WEIGHTS["hot_cold"] +
    pattern_bonus[num] * WEIGHTS["pattern"]  # 패턴에서 도출된 번호별 보정 점수
)
```

4. `pattern.py`의 결과는 직접적인 번호 점수가 아니라 조합 평가 기준이므로, 별도로 번호별 보정 점수를 도출한다:
   - 홀짝/고저 분포에서 유리한 위치의 번호에 가산점
   - 최근 연속번호 패턴에 부합하는 번호에 가산점
   - 끝자리 다양성에 기여하는 번호에 가산점

**반환값**: `dict[int, float]` — 번호별 최종 앙상블 점수 (1~45)

## 6. 몬테카를로 시뮬레이션 (montecarlo.py)

**목적**: 앙상블 점수를 확률 분포로 변환하고, 대량 시뮬레이션을 통해 최적 조합을 도출한다.

**로직**:

### 6.1 확률 변환
1. 앙상블 최종 점수를 확률로 변환한다.
   ```
   probability[num] = final_score[num] / sum(all final_scores)
   ```
2. 이 확률 분포로 numpy의 `np.random.choice`에 가중치를 적용한다.

### 6.2 시뮬레이션 루프
```
반복 (MONTE_CARLO_SIMULATIONS = 100,000회):
    1. 확률 분포에 따라 비복원 추출로 6개 번호를 뽑는다.
    2. 뽑은 조합을 패턴 기준으로 평가한다:
       - 홀짝 비율이 역대 상위 분포에 해당하는가?
       - 고저 비율이 역대 상위 분포에 해당하는가?
       - 합산값이 역대 빈출 범위 안에 있는가?
       - 끝자리 다양성이 평균 이상인가?
    3. 패턴 평가를 통과한 조합만 유효 조합으로 기록한다.
    4. 유효 조합에서 각 번호의 출현 횟수를 누적 카운트한다.
```

### 6.3 최종 번호 선정
1. 유효 조합들에서 가장 자주 등장한 번호 상위 10개를 후보군으로 선정한다.
2. 후보군 10개 중에서, 유효 조합에서 가장 많이 함께 등장한 6개 조합을 최종 추천 세트로 선정한다.
3. **보너스 번호**: 6개에 포함되지 않은 번호 중 앙상블 점수가 가장 높은 번호를 보너스로 추천한다.
4. `RECOMMEND_SETS`(5) 만큼 서로 다른 추천 세트를 생성한다.

**반환값**:
```python
[
    {"numbers": [3, 12, 17, 25, 33, 41], "bonus": 8, "confidence": 0.73},
    {"numbers": [5, 11, 22, 28, 36, 44], "bonus": 15, "confidence": 0.68},
    ...
]
```
- `confidence`: 해당 조합의 번호들이 시뮬레이션에서 출현한 비율 (참고용)

## 7. 메인 오케스트레이터 (predictor.py)

**목적**: 전체 파이프라인을 실행하고 결과를 출력한다.

### 7.1 실행 흐름

```python
def predict(db_path: str) -> dict:
    """
    1. DB 연결
    2. 4개 분석 모듈 실행 → 각각 번호별 점수 반환
    3. 앙상블 스코어링 → 번호별 최종 점수
    4. 몬테카를로 시뮬레이션 → 추천 세트 생성
    5. 결과 반환
    """
```

### 7.2 출력 형식

터미널 출력 예시:
```
============================================
🎱 로또 번호 예측 결과 (제 XXX회 기준)
============================================

📊 분석 기반: 총 XXX회차 데이터

📈 앙상블 스코어 TOP 10:
  1위: 17번 (0.892)
  2위: 33번 (0.856)
  ...

🎯 추천 세트 1 (신뢰도: 73.2%)
  번호: 03  12  17  25  33  41  + 보너스: 08

🎯 추천 세트 2 (신뢰도: 68.1%)
  번호: 05  11  22  28  36  44  + 보너스: 15

...

📋 분석 상세:
  - 빈도 분석 TOP 5: 17, 34, 12, 3, 45
  - 지연 번호 TOP 5: 8, 22, 41, 5, 19
  - 핫넘버 TOP 5: 33, 17, 25, 12, 3
  - 패턴 기준: 홀짝 3:3, 합산 100~175
============================================
```

### 7.3 CLI 인터페이스

```bash
# 기본 실행
python -m prediction.predictor --db ./lotto.db

# 옵션
python -m prediction.predictor --db ./lotto.db --sets 10 --simulations 500000
```

**CLI 옵션**:
| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `--db` | SQLite DB 파일 경로 | 필수 |
| `--sets` | 추천 세트 수 | 5 |
| `--simulations` | 몬테카를로 시뮬레이션 횟수 | 100,000 |
| `--hot-rounds` | 핫넘버 분석 최근 회차 수 | 20 |
| `--weights` | 가중치 (JSON 문자열) | config 기본값 |

## 9. 주의사항

1. **모든 점수는 0.0~1.0 정규화**: 모듈 간 점수 스케일이 다르면 가중치가 무의미해지므로, 각 모듈은 반드시 min-max 정규화를 적용한다.
2. **재현 가능성**: `np.random.seed`를 설정하여 동일 입력에 동일 결과를 보장하는 옵션을 제공한다. (기본은 랜덤)
3. **보너스 번호 처리**: 빈도 분석에서만 보너스를 0.3 가중치로 반영하고, 나머지 모듈에서는 num1~num6만 사용한다.
4. **성능**: 10만 회 시뮬레이션 기준 10초 이내 완료를 목표로 한다. numpy 벡터 연산을 활용한다.
5. **에러 처리**: DB 파일이 없거나, 데이터가 부족한 경우(최소 50회차 이상 필요) 적절한 에러 메시지를 출력한다.
